==============================
login.html 完整教學指南
==============================

【一、頁面用途】

登入頁負責：

1. 讓使用者輸入帳號密碼
2. 傳送資料給後端驗證
3. 顯示錯誤訊息


----------------------------------------
【二、後端基本寫法】
----------------------------------------

@app.route("/login", methods=["GET","POST"])
def login():

    if request.method == "POST":

        account = request.form.get("account")
        password = request.form.get("password")

        # 驗證（範例）
        if account == "admin" and password == "1234":
            return redirect("/")

        else:
            flash("帳號或密碼錯誤")
            return render_template("login.html")

    return render_template("login.html")


----------------------------------------
【三、重要規則（你這頁最重要）】
----------------------------------------

✔ 帳號欄位：永遠空白  
✔ 密碼欄位：永遠空白  

❌ 不允許：
value="{{ form.account }}"
value="{{ request.form.account }}"

👉 原因：
安全性 + 使用者體驗


----------------------------------------
【四、表單流程】
----------------------------------------

1. 使用者輸入帳密
2. 按登入
3. POST 到 /login
4. 後端驗證

成功：
→ redirect("/")

失敗：
→ flash("錯誤")
→ 重新 render


----------------------------------------
【五、錯誤訊息顯示】
----------------------------------------

Flask：

flash("登入失敗")

HTML：

{{ get_flashed_messages() }}

👉 顯示在畫面上


----------------------------------------
【六、為什麼不能保留帳號？】
----------------------------------------

1. 共用電腦會洩漏資訊
2. 不符合安全設計
3. 登入頁本來就應該乾淨


----------------------------------------
【七、你這頁的設計原則】
----------------------------------------

✔ 簡單  
✔ 安全  
✔ 無 JS  
✔ 後端控制  
✔ 無模板殘留  


----------------------------------------
【八、常見錯誤（你要避開）】
----------------------------------------

❌ 用 JS 塞資料  
❌ input 帶 value  
❌ 密碼回填  
❌ 混 fetch  

----------------------------------------
【結論】
----------------------------------------

登入頁只做一件事：

👉 驗證使用者

不要讓它變複雜
