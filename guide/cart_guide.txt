==============================
cart.html 使用與開發完整指南
==============================

【一、這個頁面在幹嘛？（先搞懂目的）】

這是一個「購物車頁面」，負責：

1. 顯示使用者加入的商品（cart_items）
2. 顯示訂單摘要（summary）
3. 提供刪除商品功能（remove_url）
4. 提供前往結帳功能（checkout_url）

👉 核心邏輯：
所有資料由「後端提供」，前端只負責顯示


----------------------------------------
【二、後端必須提供的資料（最重要）】
----------------------------------------

你後端要 render_template 時，至少要提供：

render_template(
    "cart.html",
    cart_items=...,
    summary=...,
    checkout_url=...,
    store_name=...
)

----------------------------------------
【三、cart_items 詳細結構（商品資料）】
----------------------------------------

cart_items 是「list（陣列）」，每一筆是 dict：

範例：

cart_items = [
    {
        "name": "iPhone 15",
        "price": 30000,
        "qty": 2,
        "image": "/static/img/iphone.jpg",
        "remove_url": "/cart/remove/1"
    },
    {
        "name": "AirPods",
        "price": 6000,
        "qty": 1,
        "image": None,
        "remove_url": "/cart/remove/2"
    }
]

----------------------------------------
【四、cart_items 每個欄位說明】
----------------------------------------

1. name（str）
   - 商品名稱
   - 顯示在畫面上

2. price（int / float）
   - 商品單價
   - 系統會自動轉成整數顯示（| int）

3. qty（int）
   - 商品數量

4. image（str / None）
   - 商品圖片網址
   - 如果是 None：
     → 自動使用 /static/default.png

5. remove_url（str / None）
   - 點擊「移除」按鈕時的 API
   - 如果為 None：
     → 不顯示按鈕（避免壞掉）


----------------------------------------
【五、summary（訂單摘要）】
----------------------------------------

summary 是 dict：

summary = {
    "subtotal": 66000,
    "shipping": 60,
    "discount": 1000,
    "total": 65060
}

----------------------------------------
【六、summary 每個欄位】
----------------------------------------

1. subtotal（商品總額）
2. shipping（運費）
3. discount（折扣）
4. total（最終價格）

👉 注意：
即使沒有資料，也不會壞：
因為模板寫的是：

{{ summary.xxx or 0 }}

----------------------------------------
【七、checkout_url（結帳）】
----------------------------------------

checkout_url = "/checkout"

👉 用來跳轉付款頁

如果沒有提供：
→ 按鈕不會顯示（防止錯誤）


----------------------------------------
【八、store_name（商店名稱）】
----------------------------------------

store_name = "我的商城"

👉 顯示在左上角

如果沒有提供：
→ 顯示預設：My Store


----------------------------------------
【九、畫面邏輯（你一定要懂）】
----------------------------------------

這頁有兩個狀態：

==============================
1️⃣ 有商品（cart_items 有資料）
==============================

→ 顯示：
- 商品列表
- 數量
- 價格
- 移除按鈕

==============================
2️⃣ 沒商品（cart_items 為空）
==============================

→ 顯示：
- 空狀態 UI（已設計）
- 「前往購物」按鈕

👉 不會出現：
❌ 空白頁
❌ 奇怪框
❌ 模板字

----------------------------------------
【十、圖片機制（避免破圖）】
----------------------------------------

模板寫法：

{{ item.image or '/static/default.png' }}

👉 意思：
如果 image 沒給 → 用預設圖片

👉 你需要做的：
在 static/ 放一張：

/static/default.png


----------------------------------------
【十一、刪除商品流程（重要）】
----------------------------------------

點擊「移除」：

<a href="/cart/remove/1">

👉 你後端要做：

@app.route("/cart/remove/<id>")
def remove(id):
    # 刪掉該商品
    return redirect("/cart")

👉 不要用 JS（你目前設計是純後端）


----------------------------------------
【十二、資料來源設計（實務）】
----------------------------------------

通常 cart_items 來自：

✔ session
✔ database
✔ redis（高併發）

範例（簡單版）：

cart_items = session.get("cart", [])

----------------------------------------
【十三、常見錯誤（你一定會踩）】
----------------------------------------

❌ 錯誤1：cart_items = None
→ 解法：
cart_items or []

❌ 錯誤2：summary 沒給
→ 已防呆，不會壞

❌ 錯誤3：圖片壞掉
→ 已 fallback

❌ 錯誤4：remove_url 沒給
→ 按鈕不顯示

----------------------------------------
【十四、你可以擴充的功能】
----------------------------------------

1. 數量 + - 按鈕
2. Ajax 更新數量
3. 即時價格更新
4. 優惠券系統
5. 庫存檢查
6. 登入驗證

----------------------------------------
【十五、設計原則（你這頁的核心）】
----------------------------------------

這頁的設計重點是：

✔ 不依賴前端 JS
✔ 所有邏輯由後端控制
✔ 畫面乾淨
✔ 無資料時仍像正式網站
✔ 不露模板痕跡

----------------------------------------
【結論（你要記住）】
----------------------------------------

這個頁面不是「顯示資料」

而是：

👉「讓沒有資料時，也看起來正常」

👉「讓任何資料狀況都不會壞」

👉「讓使用者感覺這是完整系統」
