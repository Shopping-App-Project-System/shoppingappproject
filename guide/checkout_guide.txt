==================================
checkout.html 完整開發教學指南
==================================

【一、頁面功能】

此頁為「訂單付款頁」，負責：

1. 收集使用者資料
2. 顯示訂單內容
3. 提交訂單給後端


----------------------------------------
【二、後端需提供資料】
----------------------------------------

render_template(
    "checkout.html",
    form=...,
    payment_methods=...,
    shipping_methods=...,
    order_items=...,
    summary=...,
    submit_url=...,
    store_name=...
)

----------------------------------------
【三、form（表單資料）】
----------------------------------------

form = {
    "name": "王小明",
    "phone": "0912345678",
    "address": "台北市",
    "note": "請晚上送"
}

👉 用途：
避免使用者填錯時資料消失


----------------------------------------
【四、payment_methods】
----------------------------------------

payment_methods = ["信用卡", "貨到付款"]

👉 用於 select 選單


----------------------------------------
【五、shipping_methods】
----------------------------------------

shipping_methods = ["宅配", "超商取貨"]

----------------------------------------
【六、order_items（訂單商品）】
----------------------------------------

order_items = [
    {"name": "iPhone", "qty": 1, "price": 30000}
]

👉 用於右側顯示訂單


----------------------------------------
【七、summary（價格）】
----------------------------------------

summary = {
    "shipping": 60,
    "total": 30060
}

👉 所有欄位都有 fallback（or 0）


----------------------------------------
【八、submit_url（送出 API）】
----------------------------------------

submit_url = "/order/create"

<form method="POST" action="/order/create">


----------------------------------------
【九、表單流程（超重要）】
----------------------------------------

1. 使用者填寫資料
2. 按「確認付款」
3. POST 到 submit_url
4. 後端驗證
5. 建立訂單
6. redirect 成功頁


----------------------------------------
【十、錯誤處理（推薦）】
----------------------------------------

若驗證失敗：

flash("資料錯誤")

return render_template(
    "checkout.html",
    form=request.form
)

👉 表單會保留原資料


----------------------------------------
【十一、畫面邏輯】
----------------------------------------

✔ 有商品 → 顯示清單  
✔ 沒商品 → 顯示空狀態  

✔ 有付款方式 → 顯示選單  
✔ 沒有 → 顯示 fallback  


----------------------------------------
【十二、常見錯誤】
----------------------------------------

❌ payment_methods = None
→ 已防呆

❌ summary 不存在
→ 不會壞

❌ order_items 空
→ 顯示空狀態


----------------------------------------
【十三、設計原則】
----------------------------------------

✔ 無 JS 依賴  
✔ 後端控制所有邏輯  
✔ 畫面乾淨  
✔ 不露模板語法  
✔ 無資料也正常顯示  


----------------------------------------
【結論】
----------------------------------------

這頁的本質是：

👉「表單 + 訂單確認」

而不是：

👉「資料展示頁」
